# エスカレーションフロー

## 自動エスカレーション

| 条件 | アクション |
|---|---|
| P1 チケットがアサインから **30 分以内**に agent が対応開始しない | lead に自動エスカレーション（Slack DM + メール） |
| P2 チケットが SLA 残り 1 時間で未解決 | lead に警告通知 |
| P3 チケットが SLA 残り 1 営業日で未解決 | agent に reminder |

## 手動エスカレーション

agent は任意のタイミングでチケットを lead にエスカレーション可能。

```
POST /api/v1/tickets/:id/escalate
{
  "reason": "ネットワーク機器の交換が必要で、自分の権限では発注できない"
}
```

## エスカレーション先の決定

1. チケットの L1 カテゴリ（IT/GA/HR）で対応チームを決定
2. 対応チームの lead 一覧から、当日のシフトに入っている lead を選択
3. lead が複数いる場合は、現在アサイン数が最小の lead を優先

## エスカレーション後のフロー

- lead がチケットを引き取り、自分で対応するか、別の agent に再アサイン
- エスカレーション履歴は `ticket_events` に記録
- SLA タイマーは停止しない（エスカレーションは SLA 一時停止の対象外）
